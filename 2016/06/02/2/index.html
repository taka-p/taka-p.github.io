続call,apply,bindの使いどころ + イテレーション系メソッド

  • apply
  • bind
  • call
  • javascript
  • イテレーター
2016-06-02

前回はcall, apply, bindの使いどころにて、call,aplly,bindの基本的な使い方やイディオムについて書きました。

今回は、イテレーション系メソッドとcall, applyを組み合わせて「テーブル操作」を行っていきます。

数値によって色分けする

DOM参照に関しては、ネイティブAPIのみ利用しています。
Resultタブから処理結果のテーブルを確認出来ます。

  • Array.prototype.slice.call(tbl.rows, 1)の部分でtr要素群の内、最初の行である項目名の行を削る
  • .forEach(function(row)でtr要素群を走査
  • 次の記述でvals, numsに、td要素群とその中の値を取り出す

ここでも最初の手順同様にtd要素群から項目名の要素を削っています。

  • ピュアな数値の配列numsを利用して、最大値と最小値を抽出

  • 最後にtd要素群であるvalsをforEachで走査して、最大値・最小値と等しい値を持つ要素に対して色を変更する行う

基本的な手順としては、下記になります。

  1. 必要なセル要素群を取得してDOM要素の配列を作成、そこから新たにデータ（数値）のみを取り出したピュアなデータ配列を作成
  2. データ配列を走査して、抽出ロジックを適用 -> 条件に適合する値を抽出
  3. DOM要素の配列を走査して、条件に適合する値に一致するDOM要素に対して走査を行う

条件によって行を非表示にする

お次は簡単、条件に該当した値を持つ行を非表示にする処理。

  • 先ほどと同様にArray.prototype.slice.call(tbl.rows, 1)の部分で、項目名の行を削る
  • ここも同様.forEach(function(row)でtr要素群を走査

  • 次の記述でvalsに、必要なtd要素群を取り出す（項目セルを削る）

  • 次の記述で条件の判定を行う（0より大きい値をもつセルはtrue）

  • 上記でtrueが返った要素を持つ行を非表示にする

条件を満たすものを抽出

最後は、条件に合致するセルを持つ行の先頭セル（項目名）を抽出するという処理です。
各商品行を走査して、100円より安い商品が存在する場合は、商品名を結合してコンソール出力しています。

  • var ret = Array.prototype.slice.call(tbl.rows, 1)の部分で不要な行を削除、処理結果をretに代入（商品名の文字列）

  • 次の記述で100より小さい値を持つ要素を抽出
    someメソッドで要素の走査を行い、該当要素の判定をfilterメソッドに受け渡して該当行の配列を作成している。

  • 上記で作成された条件に該当する行の配列から、先頭セル（商品名）の値を抽出した配列を作成

  • 上記で得た商品名の配列内の文字列群を結合した文字列を返す
    (最初に宣言したretに値が入る)

この部分に関しては、.join(', ')の方が簡潔かと思いますが、せっかくなのでreduceメソッドでイテレート。

WebAPIからデータを取得する場合

今回の記事ではWebAPIなどから、データを取得したりしないため既存のDOMから値を抽出->ロジックに適合する値を抽出->適合するDOM要素対しての操作という手順になります。

要件にもよりますが、WebAPIなどからデータを得られる場合は、下記のような手順になると思います。

  • APIからJSONを取得 -> ロジックに適合する値を抽出 -> 適合するDOM要素対しての操作
  • APIからロジック適用後のJSONを取得 -> DOM要素対しての操作
  • APIからロジック適用後のJSONを取得 -> DOM要素対しての操作 -> ユーザアクション -> ユーザアクション後のDOMから値を抽出 -> ロジックに適合する値を抽出 -> 適合するDOM要素対しての操作 -> APIにpostしてデータを更新

これについても、いつか記事にしたいと思います。
