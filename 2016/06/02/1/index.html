ネイティブのイテレーションメソッド

  • Underscore.js
  • javascript
  • lodash.js
  • イテレーター
  • イディオム
2016-06-02

Rubyの学習をしていると、「javascriptの配列・オブジェクト操作メソッド」が比較的貧弱という文脈を見かけることが多くなったため、改めてイテレーション系のメソッドを復習しました。

配列操作系の処理はUnderscore.js、lodash.jsを利用するのが定石ですが、Lodash/Underscoreは必要ない（かも）に感化され、ついでにjavascriptのビルトインメソッドも復習です。
（環境に依存せず、パフォーマンス的にも優れていると思われる）

また、イテレーションメソッド以外にもネイティブでArray,Object,Stringに実装されたメソッドのうち、Underscore/lodashでも実装されているものを紹介します。

章題はビルトインメソッド名 - Underscore/lodashメソッド名という形式で記述します。

ちなみにビルトインメソッドについて、本来はArray.prototypeもしくはObject.prototype、場合によってはString.prototypeなどの接頭辞(レシーバオブジェクト)を持ち、引数にそれらレシーバオブジェクト（データ）を処理するコールバック関数を渡します。
Array.prototype.forEach(callback)

なお、Underscore/lodashでは、_という名前空間オブジェクトからメソッドにアクセスして、引数に配列やオブジェクトなどのデータとコールバック関数を渡します。
_.each(array, callback)

イテレーション系

forEach - _.each

イテレーションメソッドの中でも一番シンプルなindex, valueを取り出すメソッドです。

オブジェクトのkey, valueを取り出す場合は下記。

ObjectはforEachを持たないためkeysメソッドでobjのプロパティ名群が格納された配列を作成して、その配列に対して呼び出す必要がある。

map - _.map

引数に渡されたコールバック関数でレシーバの配列を走査して、返された値を新しい配列に格納して返す。

every - _.every

引数に渡されたコールバック関数でレシーバの配列を走査して、全要素のコールバック判定がtureだった場合にtrueを返す。

some - _.some

引数に渡されたコールバック関数でレシーバの配列を走査して、要素のうち一つでもコールバック判定がtureだった場合にtrueを返す。

reduce - _.reduce

配列を走査する際に、コールバック関数に引数として「累積値（最終評価値）、現在の要素値」を渡せる。
また、第二引数に累積値の初期値を渡す事が出来る。

累積値の初期値を省略した場合は「array[0]が累積値・array[1]が現在の要素値」という状態で走査が始まる。


汎用的に利用出来るため、大抵の演算処理を伴う配列走査はreduceメソッドで賄えると思う。

reduceRight - _.reduceRIght

reduceメソッドは配列を左から走査するが、reduceRightメソッドはその名の通り右から走査を行う。

filter - _.filter

コールバック関数の判定がtrueだった要素群からなる、新しい配列を返す。
every, someメソッドが判定値を返すのに対して、filterメソッドは判定によって得た配列を返す。

「特定の要素を抽出する」といったケースで効果を発揮するため、リストやテーブル操作で頻繁に利用しそう。

find - _.find

コールバック関数の判定がtrueだった最初の要素を返す。
filterメソッドは判定がtrueだった要素群を配列に格納して返すのに対してfindメソッドは要素1つだけを返す。

データ構造の違いよる使い分けが想定される。

findIndex - _.findIndex

コールバック関数の判定がtrueだった最初の要素インデックスを返す。

indexOf - _.indexOf

引数に指定した要素値を持つ最初の要素インデックスを返す。
存在しない場合は-1が返る。

lastIndexOf - _.lastIndexOf

引数に指定した要素値を持つ最後の要素インデックスを返す。
存在しない場合は-1が返る。

includes - _.includes

コレクション内に指定した値があるか判定

keys - _.keys

レシーバオブジェクト自身の列挙可能なプロパティ名を列挙した配列を返す。
（enumerableがfalseのプロパティかつプロトタイプは列挙しないため、「自身のスタティック/静的メンバ、クラスメソッドを列挙する」と言い換えられる）

これは、ビルトインメソッドに追加していただいたいですね。。

helperメソッドの作成

helperとして、下記のようなメソッドを作成しても良いと思います。
（プロジェクト的にOKなら、Object.prototypeに追加しても良いかも）

reverse - _.reverce

配列内の要素を逆に並べる。
仕様を理解するよりもソースコードを見た方が直感的に理解出来る。

SQLのDESC相当処理を擬似的にクライアントで実現したいケースで使える..？

その他

isNaN - _.isNaN

NaNの判定を行うメソッド。
ただし、ネイティブのisNaNメソッドの場合、typeof 引数 == 'number'のように等値演算のロジックを含んでいるため、'hoge'のような文字列なども暗黙の型変換によりtrueと判定してしまう。
そのため、同値演算typeof 引数 === 'number'のロジックを併記するイディオムが存在する。

なお、ES6において「Underscore/lodashもしくはネイティブのisNaN+同値演算ロジック併記」に相当するロジックが実装されたNumber.isNaNが存在する。

join - _.join

引数に与えられた文字列をセパレータとして、配列要素を結合した文字列を返す。

trim - .trim

文字列の先頭と末尾から空白を削除する。

repeat - _.repeat

レシーバで受け取った文字列オブジェクトを引数に指定したn回繰り返し結合した文字列を返す。

ES7では実験的に実装されているみたいですが、まだ使うのは危なそうですね..
ネイティブでは、イディオムのArray(n + 1).join(str)を利用するのが良さそうです。


以上で、終わりです。
