識別子の命名規則

ここで紹介されていた、「一般的な識別子の記法の使い分け」についてです。

識別子の主な記法

記法 例
camelCase記法 lastName
Pascal記法 LastName

一般的には、下記のような使い分けをする。

  • 変数名や関数名 - camelCase記法
  • クラス（コンストラクタ）名 - Pascal記法

あまり意識していませんでしたが、後日改めてGoogle Javascript Style Guideなどで、世間一般的な命名や記述方法を学ぼうと思いました。

余談ですが、先日Javascript好きの知人に、自分が書いたコードをレビューしてもらったところ、変数・関数名の命名にアンダースコア記法を用いていたため「○○カスですね(^q^)」という、ありがたいお言葉を頂戴仕りました。
その知人にJavascriptの教えを請おうと思い、コンタクトを取りましたが、今の状態だと立ち直れなくなる可能性があったため、ますは復習している次第です。

数値リテラルnumber

いまのところ使った事がありませんが、数値リテラル-整数リテラルに属する「8/16進数リテラル」なるものの存在を知りました。

リテラルの階層構造は以下。

  • 数値リテラル
    • 整数リテラル
      • 10進数リテラル - 10
      • 8進数リテラル - 012
      • 16進数リテラル - 0xA
    • 浮動小数点リテラル - 10.0

なお、8進数は先頭に0、16進数は0xを付与する事で表現できる。
これによって、等価比較演算子==では、上記のリテラルもよしなに解釈してしまうため、"0x10" == 16のような条件式もtrueが返ってくる。


前置演算と後置演算

インクリメント/デクリメント演算子を記述する箇所によって、結果が変わる原因がうやむやでしたが、前置演算と後置演算の存在が原因でした。

オペランド（演算対象）に対して、前方に演算子を配置した場合は前方置換、逆に後方に演算子を配置した場合は後方置換として解釈されるらしいです。


基本型と参照型による代入の違い

Javasciprtのデータ型には基本型と参照型が存在して、array,objectは参照型。
この2種類のデータ型の挙動に違いが出るのが、代入の局面です。

上記のような差異が発生する原因は、基本型はy = xの時にメモリ領域を新たに確保しているが、参照型はarr2 = arr1の時に、arr1のデータが格納される「メモリアドレス」をarr2にコピーしているからです。

ちなみに、このような値の渡し方を「参照渡し」と呼ぶらしいです。

ifとswitchの使い分け

Javascriptでは条件分岐の制御構文としてifとswitchを用意していますが、いまいち使い分けの境界が理解できていなかったので、この機会に明記します。

  • if - 単純な条件の場合はこちら
  • switch - 複雑な条件の場合はこちら

ざっくりとですが、if文の場合は、処理を記述する条件ブロックの都度、条件式を記述する必要があります。
同一の変数に対して比較を行う場合は、構文の上部にのみ条件式を定義するswitch文を利用する事で、並列の関係にある条件値をキレイに表現できます。


forとwhileの使い分け

forとwhileについても、使いどころがいまいち理解できていませんでした。

for文では、「初期化式、ループ継続条件式、増減式」という式を設定しますが、これらの式のいずれかが複雑になる場合はwhileを利用した方がキレイに記述できます。
例えば、初期化する変数が多い、といった理由が挙げられます。

また、do…while文とfor…inの使いどころについても、迷いがちだったため明記します。

  • do…while - 条件に関わらず、1回は処理を実行したい場合（条件式が処理ブロックの後に配置されるため）
  • for…in - オブジェクト（連想配列、ハッシュ）のプロパティを取り出したい場合

for…inの例のみ下記に示します。


例外処理try…catch…finaly

Javascriptにも例外処理があった事を初めて知りました_:(´ཀ`」 ∠):_

  • try - 例外が発生するかもしれない処理
  • catch (e) - 例外が発生した場合の処理
  • finaly - 例外の有無にかかわらず実行する処理
  • throw new Error(message) - tryにおいて、強制的に例外を発生させる

直近で、「想定する値以外が渡ってきた場合」という処理をif文で実装していたのを思い出したため、イメージが掴み易かったです。

なお、finalyの使いどころはパッと浮かびませんでした…
