リンク関連のビューヘルパー

アンカータグ、URL文字列の生成を行うヘルパーなどがある。

  • link_toメソッド
    アンカータグを生成する。

body: リンクテキスト、url: リンク先のURL、html_opt: 動作オプション

オプション 概要
method HTTPメソッドの種類を便宜的に付与する※
remote Ajaxでリンクを処理する
data 確認ダイアログを表示するconfirm、クリック後に日アクティブな状態であることを表示するdisable_withなど、javascript機能を付与する
属性名 id,clas,styleなどの一般的なHTML属性

※本来、ブラウザはGET,POSTのみを受け付けるが、DELETE,PUT,PATCHのようなメソッドをRESTful思想に基づき、便宜的に付与している


  • url_forメソッド
    引数に与えられたオプションに基づいて、URLを生成する。
    link_toメソッドのアンカータグを生成しないバージョン。
    （link_toメソッドの引数指定は、url_forメソッドのそれに基づく）

opts: URLの生成オプション

オプション 概要
controller コントローラー名
action アクション名
host ホスト名
protocol プロトコル名
anchor アンカー名
only_path 相対URLを返す
trailing_slash 末尾にスラッシュを付与
user HTTP認証に使用するユーザー名
password HTTP認証に使用するパスワード


外部リソース指定のビューヘルパー

各種静的リソースを呼び出すためのヘルパー。

  • image_tagメソッド
    <img>タグの生成

src: 画像フォルダのパス、opts: 動作オプション
なおsrcには、絶対パスもしくわapp/assets/imagesフォルダからの相対パスを指定する


  • path_to_xxxメソッド
    引数に指定したオプションに基づき、URLを生成する。
    タグを生成しないバージョン。
メソッド 概要
path_to_javascript(src) jsのパスを取得
path_to_stylesheet(src) スタイルシートのパスを取得
path_to_image(src) 画像のパスを取得
path_to_audio(src) 音声のパスを取得
path_to_video(src) 動画のパスを取得


その他のビューヘルパー

その他の便利系メソッドを抜粋。

  • debugメソッド
    引数に指定したオブジェクトをYAML形式で表示する。

object: 出力対象のオブジェクト


  • captureメソッド
    テンプレートの一部を変数に格納する。
    部分テンプレート化する粒度では無い場合に用いる。

@var: 変数、template: 変数に格納する任意のテンプレート

共通レイアウトを定義・利用する

ここは、少しつまづきました。
以下、概念的な部分に重きを置くため、文章中心になります。

Railsにおいて「共通レイアウト」を定義・利用する方法は大きく以下の二つに分類される。

  1. ヘッダーやフッターなど「コンテンツの外枠」の共通パーツを定義するレイアウト/レイアウトテンプレート
  2. 「コンテンツ内」の部分的な共通パーツを定義する部分テンプレート

レイアウトは広域に利用される部品を定義、部分テンプレートは局所的な部品を定義する。
とも言い換えられるかと思います。

多くのフレームワークでは、2の部分テンプレートによって、1のレイアウトもカバーするケースが多いと思いますが（知らないだけかも）、Railsにおいては1のレイアウトという機能も利用されるようです。

まだ、基礎学習のレベルなので確かではありませんが、レイアウトはControllerによるレイアウト管理に用いられ、部分テンプレートはViewの中で部品を管理するために用いられている。
と、一旦解釈しています。
